## SOP概述

### 1.1 目的

本SOP旨在规范内容审核与平台风控服务的全流程管理，确保审核质量稳定高效，识别并防范各类平台风险，保护用户安全，维护平台生态健康，提升客户满意度。通过标准化的作业流程，实现审核准确率≥99.5%、风险响应时效≤5分钟、漏审率≤0.3%的核心目标。

### 1.2 适用范围

本SOP适用于各类平台的内容审核与风控服务，包括但不限于：
- 社交平台内容审核
- 电商平台内容审核
- 短视频/直播平台内容审核
- 新闻资讯平台内容审核
- 游戏社区内容审核
- 其他UGC/PGC内容平台

服务内容涵盖：文本审核、图片审核、视频审核、音频审核、直播审核、用户行为风控、账号安全管理等。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| UGC | 用户生成内容（User Generated Content） |
| PGC | 专业生成内容（Professional Generated Content） |
| 初审 | 机器/AI自动审核 |
| 复核 | 人工二次审核 |
| 申诉 | 用户对审核结果提出异议 |
| 风控 | 风险识别与控制 |
| 白名单 | 优先审核通过的内容/账号 |
| 黑名单 | 禁止发布内容的账号 |
| 敏感词 | 违规词汇库 |
| 准确率 | 正确审核内容占比 |
| 漏审率 | 违规内容未被识别占比 |
| 误审率 | 正常内容被误判违规占比 |
| 召回率 | 识别违规内容占实际违规内容比例 |
| KPI | 关键绩效指标 |
| SLA | 服务水平协议 |

### 1.4 核心KPI指标

| 指标 | 目标值 | 测量方式 | 权重 |
|------|--------|----------|------|
| 审核准确率 | ≥99.5% | 质量抽检统计 | 30% |
| 漏审率 | ≤0.3% | 漏审案例统计 | 25% |
| 误审率 | ≤0.5% | 误审案例统计 | 15% |
| 平均审核时效 | ≤2分钟 | 审核时长统计 | 10% |
| 用户申诉成功率 | ≥98% | 申诉处理统计 | 10% |
| 风险响应时效 | ≤5分钟 | 风险事件响应 | 10% |
| 按时交付率 | 100% | 项目计划对比 | 5% |
| 客户满意度 | ≥4.8分（5分制） | 客户评价统计 | 5% |

## 客户对接与项目启动

### 2.1 客户需求调研

#### 2.1.1 调研流程（T-15至T-10天）

**步骤1：初次接触**
- 销售顾问与客户初次沟通，了解基本情况
- 填写《内容审核需求调研表》（见附录A）
- 收集客户平台类型、用户规模、内容类型等基础信息

**步骤2：深度调研**
- 风控专家上门或线上进行深度调研
- 填写《客户需求调研表》（见附录B）
- 调研内容包括：
  - 平台基本情况：平台类型、日活用户、日均内容量
  - 审核需求：内容类型、审核维度、违规分类
  - 风控需求：风险类型、风控规则、响应时效
  - 历史情况：过往违规案例、现有审核流程
  - 特殊要求：行业特定规范、合规要求

**步骤3：需求评估**
- 分析客户现有审核流程和痛点
- 评估客户审核标准与行业标准的差异
- 填写《需求评估报告》

#### 2.1.2 调研清单

| 调研项目 | 具体内容 | 收集方式 |
|----------|----------|----------|
| 平台资质 | 营业执照、ICP备案、行业资质 | 复印件或照片 |
| 内容数据 | 过往3个月违规内容案例 | 案例库 |
| 审核标准 | 现有审核规则、分类标准 | 文档 |
| 风控规则 | 风险识别规则、处置流程 | 文档 |
| 系统对接 | 现有审核系统、API接口文档 | 技术文档 |
| 人员情况 | 现有审核团队、培训情况 | 访谈 |

### 2.2 服务方案制定

#### 2.2.1 方案制定流程（T-10至T-7天）

**步骤1：服务模式确定**
根据客户需求和规模，确定服务模式：
- **基础服务**：机器初审+人工复核
- **标准服务**：基础服务+风控管理+质量监控
- **高级服务**：标准服务+AI模型定制+专项风控
- **定制服务**：根据客户特殊需求定制

**步骤2：团队配置**
- 确定项目负责人：具有3年以上内容审核经验
- 确定审核主管：具有2年以上团队管理经验
- 确定审核员：根据日均内容量配置
- 确定风控专员：负责风险识别与处置
- 确定质量专员：负责质量监控与抽检

**步骤3：服务方案编制**
方案书包含以下内容：
- 客户需求分析
- 服务范围与内容
- 服务团队配置
- 审核标准体系
- 风控管理方案
- 服务流程与时间节点
- KPI指标体系
- 服务收费标准
- SLA服务水平协议
- 保密条款

**步骤4：方案沟通与确认**
- 向客户汇报服务方案
- 听取客户意见，调整方案
- 双方确认最终方案

### 2.3 合同签订与项目启动

#### 2.3.1 合同签订流程（T-7至T-5天）

**步骤1：合同起草**
- 使用标准合同模板（见附录D）
- 根据客户情况调整合同条款
- 明确服务内容、费用、付款方式、违约责任

**步骤2：合同审核**
- 法务部门审核合同条款
- 财务部门审核费用条款
- 项目负责人审核服务内容

**步骤3：合同签署**
- 双方确认合同无误
- 打印合同，双方签字盖章
- 合同一式两份，双方各执一份

**步骤4：款项收取**
- 客户支付首期服务费
- 开具发票并交付客户
- 确认款项到账

#### 2.3.2 项目启动流程（T-5至T日）

**步骤1：项目启动会议**
- 时间：服务启动前1天
- 参与人：客户负责人、项目负责人、服务团队
- 内容：
  - 明确各方职责分工
  - 确认服务范围和KPI指标
  - 确定沟通机制和频率
  - 确定紧急联系人

**步骤2：服务准备**
- 配置审核系统和工具
- 准备审核标准和案例库
- 准备工作文档模板
- 设置工作提醒和日历

**步骤3：正式启动**
- 服务正式开始
- 按约定流程开展工作
- 第一周每日与客户沟通进度

### 2.4 审核标准制定

#### 2.4.1 标准制定原则

**原则1：合法合规**
- 符合国家法律法规
- 符合行业监管要求
- 符合平台用户协议

**原则2：明确具体**
- 违规分类清晰明确
- 判断标准具体可操作
- 边界案例有明确指引

**原则3：动态更新**
- 定期更新审核标准
- 及时响应新违规类型
- 持续优化审核规则

#### 2.4.2 标准制定流程

**步骤1：收集素材**
- 收集客户现有审核标准
- 收集行业通用审核标准
- 收集历史违规案例

**步骤2：分类梳理**
- 违规内容分类
- 风险等级划分
- 审核细则制定

**步骤3：案例编写**
- 正例编写：合规内容示例
- 反例编写：违规内容示例
- 边界案例：特殊情况处理指引

**步骤4：评审确认**
- 内部评审
- 客户确认
- 标准定稿

### 2.5 系统对接与测试

#### 2.5.1 系统对接流程

**步骤1：技术评估**
- 评估客户系统架构
- 确定对接方式（API/SDK/其他）
- 制定技术方案

**步骤2：接口开发**
- 开发对接接口
- 编写接口文档
- 进行接口测试

**步骤3：联调测试**
- 与客户系统联调
- 测试数据传输
- 测试审核回调

**步骤4：上线验收**
- 上线前测试
- 小规模试运行
- 正式验收

#### 2.5.2 对接文档

对接文档包含：
- 接口说明
- 数据格式
- 错误码说明
- 调用示例

## 内容审核标准体系

### 3.1 违规内容分类

#### 3.1.1 一级违规（严重）

| 违规类型 | 定义 | 处置措施 |
|----------|------|----------|
| 政治敏感 | 反对宪法确定的基本原则、危害国家安全、破坏国家统一、损害国家荣誉利益、宣扬恐怖主义极端主义、煽动民族仇恨歧视等 | 立即删除、账号封禁、上报监管部门 |
| 色情低俗 | 露骨色情描述、人体隐私部位暴露、性挑逗语言、低俗暗示等 | 立即删除、账号封禁、内容下架 |
| 暴力恐怖 | 宣扬暴力、血腥恐怖内容、教唆犯罪、非法持有枪支毒品等 | 立即删除、账号封禁、上报监管部门 |
| 诈骗违法 | 网络诈骗、虚假广告、非法金融活动、侵犯个人隐私等 | 立即删除、账号封禁、冻结资金、上报监管部门 |
| 侵权盗版 | 侵犯知识产权、盗用他人内容、侵犯肖像权名誉权等 | 删除内容、下架处理、协助维权 |

#### 3.1.2 二级违规（中等）

| 违规类型 | 定义 | 处置措施 |
|----------|------|----------|
| 虚假广告 | 夸大宣传、虚假承诺、误导消费者等 | 删除内容、警告、限流 |
| 恶意营销 | 刷屏广告、垃圾营销、诱导分享关注等 | 删除内容、警告、限流 |
| 低俗骚扰 | 言语骚扰、低俗玩笑、不尊重他人等 | 删除内容、警告、限流 |
| 谣言不实 | 传播不实信息、制造恐慌、歪曲事实等 | 删除内容、警告、澄清 |
| 恶意攻击 | 人身攻击、侮辱谩骂、网络暴力等 | 删除内容、警告、限流 |

#### 3.1.3 三级违规（轻微）

| 违规类型 | 定义 | 处置措施 |
|----------|------|----------|
| 垃圾信息 | 无意义内容、重复内容、刷屏灌水等 | 删除内容、警告 |
| 标题党 | 夸张标题、内容不符、误导点击等 | 内容修正、警告、限流 |
| 不友善言论 | 言语不当、容易引发争议等 | 提醒修改、警告 |
| 违规配图 | 配图与内容不符、使用违规图片等 | 删除配图、警告 |

### 3.2 审核判断细则

#### 3.2.1 政治敏感判断细则

**判断标准1：敏感人物与事件**
- 禁止出现国家领导人不当调侃、歪曲
- 禁止出现敏感历史事件不当解读
- 禁止出现分裂国家言论

**判断标准2：意识形态安全**
- 禁止宣扬西方错误思潮
- 禁止攻击社会主义制度
- 禁止历史虚无主义

**判断标准3：民族宗教**
- 禁止煽动民族仇恨
- 禁止侮辱宗教信仰
- 禁止破坏民族团结

#### 3.2.2 色情低俗判断细则

**判断标准1：露骨色情**
- 人体隐私部位直接暴露
- 性行为直接描述或展示
- 色情场所推广

**判断标准2：低俗暗示**
- 性挑逗语言
- 低俗肢体动作
- 暗示性场景

**判断标准3：软色情**
- 擦边球内容
- 低俗趣味
- 过度消费身体

#### 3.2.3 暴力恐怖判断细则

**判断标准1：血腥暴力**
- 血腥暴力画面
- 虐待动物画面
- 暴力犯罪展示

**判断标准2：恐怖极端**
- 恐怖组织标识
- 恐怖袭击宣传
- 极端主义思想

**判断标准3：教唆犯罪**
- 犯罪方法教授
- 犯罪工具展示
- 犯罪行为鼓励

#### 3.2.4 诈骗违法判断细则

**判断标准1：网络诈骗**
- 虚假中奖信息
- 冒充客服诈骗
- 投资理财诈骗

**判断标准2：虚假广告**
- 夸大产品效果
- 虚假用户评价
- 假冒伪劣商品

**判断标准3：非法金融**
- 非法集资
- 虚拟货币炒作
- 非法借贷

### 3.3 审核结果定义

#### 3.3.1 审核结果分类

| 结果类型 | 定义 | 后续操作 |
|----------|------|----------|
| 通过 | 内容合规，无违规问题 | 正常发布 |
| 删除 | 内容违规，需要删除 | 删除内容、记录违规 |
| 限流 | 内容存在一定问题，限制推荐 | 减少推荐、观察后续 |
| 修改 | 内容部分违规，修改后可通过 | 退回修改、重新审核 |
| 封禁 | 严重违规，禁止发布 | 删除内容、封禁账号 |

#### 3.3.2 结果标记标准

**标记内容**
- 违规类型
- 违规等级
- 违规原因
- 处置建议
- 审核人
- 审核时间

### 3.4 审核尺度把控

#### 3.4.1 尺度把握原则

**原则1：从严把控**
- 政治敏感零容忍
- 色情暴力零容忍
- 诈骗违法零容忍

**原则2：合理审慎**
- 区分无意和故意
- 区分首次和惯犯
- 区分情节轻重

**原则3：标准统一**
- 相同情况相同处理
- 前后标准保持一致
- 不同审核员标准统一

#### 3.4.2 特殊情况处理

**情况1：模棱两可**
- 提交小组讨论
- 参考过往案例
- 请示主管决定

**情况2：新型违规**
- 记录案例
- 更新标准
- 培训团队

**情况3：紧急情况**
- 先行处置
- 后续补录
- 及时汇报

### 3.5 审核案例库建设

#### 3.5.1 案例库结构

**结构1：违规分类**
- 按违规类型分类
- 按违规等级分类
- 按内容类型分类

**结构2：案例内容**
- 案例截图/文本
- 违规原因分析
- 处置结果记录
- 审核要点提示

**结构3：案例索引**
- 关键词索引
- 类型索引
- 时间索引

#### 3.5.2 案例库更新

**更新频率**
- 每周新增案例
- 每月案例整理
- 每季度案例回顾

**更新流程**
- 案例收集
- 案例整理
- 案例审核
- 案例入库
- 案例培训

**规则类型2：图片识别**
- 配置违规图片模型
- 设置识别阈值

**规则类型3：音频识别**
- 配置语音识别模型

**规则类型4：视频识别**

- 色情暴力次高优先级

#### 4.2.1 敏感词库管理

- 暴力恐怖词库
- 广告营销词库
- 其他违规词库

**库更新**
- 定期更新词库
- 及时新增敏感词
- 清理无效词

#### 4.2.2 过滤流程

### 附录E：常见问题解答（FAQ）

**Q1：审核过程中遇到不确定的案例怎么办？**
A：标记为不确定，提交小组讨论，请示主管决定，参考过往案例。

**Q2：发现新型违规类型怎么办？**
A：记录案例，及时上报，更新审核标准，培训团队。

**Q3：用户对审核结果不认可怎么办？**
A：引导用户走申诉流程，耐心解释审核标准，客观公正处理申诉。

**Q4：审核任务积压怎么办？**
A：增加人员投入，延长工作时间，优化审核流程，提高自动化程度。

**Q5：审核员审核质量不达标怎么办？**
A：分析原因，重新培训，加强审核，必要时调整岗位。

**Q6：发现重大风险怎么办？**
A：立即处置，及时上报，保存证据，配合调查。
