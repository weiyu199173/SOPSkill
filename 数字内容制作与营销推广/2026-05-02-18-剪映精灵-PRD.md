# 产品需求文档 (PRD)

## 剪映精灵 - 短视频AI智能剪辑与批量发布工具

> **文档版本**: v1.0  
> **创建时间**: 2026-05-02  
> **痛点来源**: B站、抖音、小红书用户痛点采集  
> **创业价值评分**: 7.55/10（6维度加权）

---

## 1. 项目背景

### 1.1 痛点发现

通过WebSearch从B站、抖音、小红书等平台采集用户痛点数据，发现**短视频创作者日更剪辑压力大**是视频创作者群体中最突出的效率痛点。

**典型用户反馈**：
- *"深夜剪完视频，强撑着困意一个个上传，反复填写标题、标签、简介，还要切换账号发布"* —— B站UP主
- *"传统总耗时10-14小时，AI辅助后2-2.5小时"* —— 创作者工具评测
- *"每天只需花1小时就能轻松实现日更，把剩下的时间全部用来打磨创意"* —— 短视频博主期望

### 1.2 市场规模

- **受影响用户**: 300万+ B站UP主、抖音创作者、短视频博主、MCN机构
- **付费意愿**: 高（剪辑工具月费50-300元，AI工具更高）
- **痛点频次**: 每天（日更创作者占比超40%，周更占比超80%）

### 1.3 竞品分析

| 竞品 | 优势 | 不足 | 月费 |
|------|------|------|------|
| 剪映 | 免费、功能全面、抖音生态 | 自动化程度低、需大量手动操作 | 免费 |
| Recapo.ai | 对话式剪辑、全流程自动化 | 价格较高、仅支持特定格式 | 99-299元 |
| 必剪 | B站生态、免费 | 功能较基础 | 免费 |
| Descript | AI剪辑领先 | 英文为主、国内访问慢 | $24-33/月 |

**核心差异化机会**: 现有工具**缺乏"从素材到发布"的全流程自动化**能力，创作者仍需在多个工具间切换。市场需要一款**AI驱动的端到端视频生产工具**，覆盖素材整理→智能剪辑→自动字幕→多平台发布的完整链路。

## 3. 核心用户故事

### US-1: AI智能剪辑
> 作为一名B站UP主，我上传原始录像素材后，希望AI能自动识别精彩片段、去除口误和停顿、添加转场效果，将2小时的原始素材剪辑为10分钟的高质量成片，这样我可以把剪辑时间从6小时缩短到30分钟。

### US-2: 自动字幕生成
> 作为短视频创作者，我希望系统能自动识别视频中的语音并生成精准字幕，支持中英文混排、自动断句、关键词高亮，省去我手动添加字幕的2小时。

### US-3: 长视频智能切片
> 作为知识分享者，我有一场2小时的直播录像，希望AI能自动识别话题切换点，将长视频切片为20-30个1-3分钟的短视频，每个切片都有完整的开头和结尾，可以直接发布到抖音和小红书。

### US-4: 多平台批量发布
> 作为日更创作者，我每天需要将视频发布到B站、抖音、快手、小红书4个平台。我希望剪辑完成后，系统能自动生成各平台要求的封面图、标题、标签，并按设定时间自动发布。

## 5. 主流程

### 5.1 核心业务流程

```
用户上传素材(视频/音频/图片)
    ↓
AI分析素材(语音识别/场景检测/精彩片段标记)
    ↓
用户选择剪辑模式:
  ├── 智能剪辑(AI全自动)
  ├── 模板剪辑(选择预设模板)
  └── 手动剪辑(AI辅助)
    ↓
AI执行剪辑 → 自动添加字幕 → 应用转场/特效
    ↓
用户预览 → 微调(可选)
    ↓
选择导出格式 → 多平台适配(封面/标题/标签)
    ↓
一键发布(或定时发布) → 数据追踪
```

### 5.2 AI智能剪辑流程

```
输入: 原始视频素材(1-3小时)
    ↓
Step 1: 语音识别(Whisper API)
  → 生成完整字幕文本 + 时间轴
  → 识别语言(中文/英文/混合)
    ↓
Step 2: 内容分析
  → 口误/重复/停顿检测(静音>2秒、语气词"嗯/啊/那个")
  → 话题分段(语义变化检测)
  → 高光片段评分(语速变化/音量峰值/关键词命中)
    ↓
Step 3: 智能剪辑
  → 去除口误和冗余停顿
  → 保留高光片段(按评分排序)
  → 自动添加转场(淡入淡出/推拉)
  → 背景音乐匹配(可选)
    ↓
Step 4: 字幕生成
  → 时间轴精确对齐
  → 自动断句(每行≤15字)
  → 关键词高亮(可选)
    ↓
输出: 剪辑后视频 + SRT字幕文件
```

## 8. 异常场景与边界条件

| 场景 | 处理策略 |
|------|----------|
| 视频格式不支持 | 自动转码为MP4(H.264)，不支持时提示用户 |
| 语音识别准确率低 | 支持用户手动编辑字幕，提供纠错建议 |
| AI剪辑效果不满意 | 保留原始素材，支持回退和手动调整 |
| 处理超时(视频过长) | 分段处理，显示进度条，支持后台处理 |
| 平台发布失败 | 记录原因，支持重试，视频本地保留 |
| 存储空间不足 | 提前预警，支持清理已处理素材 |
| 并发处理压力 | 任务队列限流，优先处理付费用户 |
| 网络中断 | 断点续传，已处理部分保留 |

## 9. 技术方案概要

### 9.1 技术选型

| 层级 | 技术方案 | 理由 |
|------|----------|------|
| 前端 | Next.js + React + TailwindCSS | 视频预览、时间轴编辑器 |
| 视频处理 | FFmpeg (WASM/Server) | 行业标准视频处理库 |
| 语音识别 | OpenAI Whisper API | 中文识别准确率高 |
| AI推理 | Python + FastAPI | 视频分析、内容理解 |
| 对象存储 | Cloudflare R2 / AWS S3 | 视频文件存储，成本低 |
| 消息队列 | Redis Streams | 异步任务处理 |
| 部署 | Vercel (前端) + Railway (后端) | 低成本、自动扩缩容 |

### 9.2 系统架构

```
[Web前端(视频编辑器)]
    ↓
[API网关] → [素材服务] → [对象存储(S3/R2)]
    ↓
[AI处理队列]
    ├── [语音识别服务(Whisper)]
    ├── [内容分析服务]
    └── [视频剪辑服务(FFmpeg)]
    ↓
[字幕生成服务] → [导出服务]
    ↓
[发布服务]
    ├── [B站API]
    ├── [抖音API]
    ├── [快手API]
    └── [小红书API]
    ↓
[数据采集服务] → [数据看板]
```

## 10. 验收标准

### 10.1 功能验收

- [ ] 用户可注册/登录，绑定至少1个视频平台账号

- [ ] 页面首屏加载 < 2秒
- [ ] 视频预览延迟 < 500ms
- [ ] 支持20个并发视频处理任务

### 10.3 安全验收

- [ ] 各平台OAuth授权流程合规
- [ ] 用户视频文件加密存储
- [ ] 处理完成后原始素材可安全删除
