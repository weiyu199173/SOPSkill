# 产品需求文档 (PRD)

## 一键分发Pro - 自媒体多平台智能内容分发工具

> **文档版本**: v1.0  
> **创建时间**: 2026-05-02  
> **痛点来源**: 知乎、小红书、B站、今日头条用户痛点采集  
> **创业价值评分**: 7.85/10（6维度加权）

---

## 1. 项目背景

### 1.1 痛点发现

通过WebSearch从知乎、小红书、B站、今日头条等平台采集用户痛点数据，发现**自媒体多平台内容分发效率极低**是创作者群体中呼声最高的痛点之一。

**典型用户反馈**：
- *"每天光是切换不同平台账号登录就要花半小时，发布一篇文章得重复上传十几次"* —— 小红书创作者
- *"凌晨两点还在调整公众号排版，小红书、抖音、公众号三个平台要发三种格式"* —— 自媒体运营
- *"好不容易做完的图被甲方要求再改一版"* —— 内容营销人员

### 1.2 市场规模

- **受影响用户**: 500万+ 自媒体创作者、UP主、博主、内容营销团队
- **付费意愿**: 高（现有工具月费50-200元，用户仍觉值得）
- **痛点频次**: 每天（日均发布1-5篇内容的创作者占比超60%）

### 1.3 竞品分析

| 竞品 | 优势 | 不足 | 月费 |
|------|------|------|------|
| 媒介集市 | 性价比高、支持主流平台 | 功能较基础，缺乏AI适配 | 50-100元 |
| 新榜小豆芽 | 50+平台支持 | 价格较高、操作复杂 | 100-200元 |
| 融媒宝 | 界面直观 | 免费版限制多 | 50-150元 |
| 易媒助手 | 数据分析强 | 发布稳定性一般 | 80-180元 |

**核心差异化机会**: 现有工具均缺乏**AI智能格式适配**能力，用户仍需手动调整各平台格式。

| 用户类型 | 特征 | 核心需求 |

| **自媒体工作室** | 5-20人团队，管理10+账号，日更5-10篇 | 批量操作、团队协作、数据分析 |

### 2.2 用户规模估算

- 个人创作者: ~400万
- 自媒体工作室: ~50万
- 中小企业新媒体: ~50万
- **总计TAM**: ~500万用户

## 3. 核心用户故事

### US-1: 个人创作者一键分发
> 作为一名小红书+公众号+B站三平台创作者，我希望写完一篇文章后，系统能自动将内容适配为各平台格式并一键发布，这样我每天可以节省2-3小时的重复操作时间。

### US-2: 工作室批量管理
> 作为自媒体工作室运营，我希望能一次性导入10篇待发布内容，设置各平台发布时间后自动排队发布，并查看全平台数据汇总报表。

### US-3: 智能格式适配
> 作为内容创作者，我希望粘贴一篇Markdown格式的文章后，系统能自动转换为小红书图文笔记格式（含封面图生成）、公众号排版格式、B站专栏格式，无需我手动调整。

### US-4: 数据驱动优化
> 作为运营人员，我希望在发布后能自动收集各平台的阅读量、点赞、评论等数据，生成对比报表，帮助我判断哪类内容在哪个平台表现更好。

## 5. 主流程

### 5.1 核心业务流程

```
用户登录 → 创建内容(Markdown编辑器)
    ↓
选择目标平台(多选) → AI自动格式适配
    ↓
预览各平台效果 → 确认/手动微调
    ↓
设置发布时间(立即/定时) → 一键发布
    ↓
发布状态追踪 → 数据自动采集
    ↓
数据看板查看 → 优化建议
```

### 5.2 AI格式适配流程

```
输入: Markdown内容 + 目标平台列表
    ↓
解析内容结构(标题/正文/图片/标签)
    ↓
平台规则引擎匹配:
  - 小红书: 标题≤20字, 正文分段+emoji, 图片3:4, 标签#格式
  - 公众号: 富文本HTML, 封面图2.35:1, 摘要
  - B站: 专栏Markdown, 封面图16:9
  - 知乎: Markdown, 话题标签
  - 今日头条: 标题≤30字, 封面图3:2
    ↓
生成各平台适配版本 → 用户预览确认
```

## 7. MVP范围界定

### 7.1 MVP包含

- ✅ Web端内容编辑器（Markdown）
- ✅ 6大平台账号绑定与OAuth授权
- ✅ AI格式自动适配（标题/正文/图片/标签）
- ✅ 一键发布（支持6大平台）
- ✅ 定时发布队列
- ✅ 基础数据看板

### 7.2 MVP不包含

- ❌ 移动端App（后续迭代）
- ❌ AI内容生成（v2.0）
- ❌ AI封面图生成（v2.0）
- ❌ 团队协作与权限管理（v2.0）
- ❌ 竞品监控（v3.0）
- ❌ 付费订阅系统（MVP期间免费使用）

## 8. 异常场景与边界条件

| 场景 | 处理策略 |
|------|----------|
| 平台API限流 | 自动排队重试，指数退避策略，最多重试3次 |
| 平台OAuth Token过期 | 提前刷新Token，过期后引导用户重新授权 |
| 发布失败 | 记录失败原因，通知用户，支持手动重试 |
| 图片格式不支持 | 自动转换为平台支持的格式（WebP/JPG/PNG） |
| 内容超长 | 自动截断并提示用户，支持手动调整 |
| 网络异常 | 本地缓存草稿，网络恢复后自动同步 |
| 并发发布冲突 | 使用消息队列串行处理，保证发布顺序 |

## 9. 技术方案概要

### 9.1 技术选型

| 层级 | 技术方案 | 理由 |
|------|----------|------|
| 前端 | Next.js + React + TailwindCSS | SSR/SSG支持、开发效率高 |
| 编辑器 | TipTap (ProseMirror) | Markdown富文本、可扩展 |
| 后端 | Node.js + Express | 轻量高效、生态丰富 |
| 数据库 | PostgreSQL + Redis | 关系数据持久化 + 缓存/队列 |
| AI服务 | OpenAI API / 豆包API | 内容格式智能适配 |
| 部署 | Vercel (前端) + Railway (后端) | 低成本、自动扩缩容 |

### 9.2 系统架构

```
[Web前端] → [API网关] → [内容服务] → [AI适配引擎]
                              ↓
                         [发布队列(Redis)]
                              ↓
                    ┌─────────┼─────────┐
               [微信API] [小红书API] [B站API] [知乎API] [头条API]
                              ↓
                         [数据采集服务] → [数据看板]
```

## 10. 验收标准

### 10.1 功能验收

- [ ] 用户可注册/登录，绑定至少1个平台账号
- [ ] 编辑器支持Markdown输入，实时预览
- [ ] 选择目标平台后，AI在5秒内完成格式适配
- [ ] 适配结果准确率 > 90%（标题字数、图片尺寸、标签格式）
- [ ] 一键发布到已授权平台，成功率 > 95%
- [ ] 定时发布按设定时间准确执行（误差 < 1分钟）
- [ ] 数据看板在发布后24小时内展示基础数据

### 10.2 性能验收

- [ ] 页面首屏加载 < 2秒（LCP）
- [ ] AI格式适配响应 < 5秒
- [ ] 支持100个并发用户同时操作

### 10.3 安全验收

- [ ] OAuth授权流程符合各平台规范
- [ ] 用户密码bcrypt加密存储
- [ ] API密钥加密存储，日志脱敏
