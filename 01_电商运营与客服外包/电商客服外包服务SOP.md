## SOP概述

### 1.1 目的
本SOP旨在规范电商客服外包服务的全流程管理，确保服务质量稳定、高效，满足客户需求，提升客户满意度和转化率。

### 1.2 适用范围
本SOP适用于所有电商平台（淘宝、天猫、京东、拼多多、抖音、小红书等）的客服外包服务。

### 1.3 术语定义
| 术语 | 定义 |
|------|------|
| KPI | 关键绩效指标 |
| AHT | 平均处理时长 |
| FCR | 首次解决率 |
| CSAT | 客户满意度 |
| SLA | 服务水平协议 |
| VOC | 客户之声 |

### 1.4 核心KPI指标
| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 3分钟回复率 | ≥95% | 系统自动统计 |
| 平均响应时长 | ≤30秒 | 系统自动统计 |
| 首次解决率 | ≥90% | 工单+质检统计 |
| 客户满意度 | ≥4.8分（5分制） | 评价统计 |
| 咨询转化率 | 提升10%以上 | 订单统计 |
| 差评处理率 | 100% | 差评记录统计 |

## 服务对接与启动流程

### 2.1 客户对接阶段（T-15至T-7天）

**步骤1：需求调研**
- 填写《客户需求调研表》（见附录A）
- 了解客户店铺基本信息、商品信息、平台规则
- 明确客户服务要求、KPI指标、特殊需求
- 收集客户历史客服数据、话术、产品资料

**步骤2：方案制定**
- 根据需求制定《服务方案书》
- 确定团队配置、服务模式、价格方案
- 制定KPI指标体系
- 形成《SLA服务水平协议》

**步骤3：合同签订**
- 双方确认合同条款
- 签订《客服外包服务合同》
- 客户支付首期款项

### 2.2 服务准备阶段（T-7至T-1天）

**步骤1：资源配置**
- 确定项目负责人、客服主管、客服团队
- 配置电脑、网络、账号、系统权限
- 部署客服系统、工单系统、质检系统

**步骤2：资料收集**
- 收集商品资料：产品参数、规格、价格、库存
- 收集店铺规则：退换货政策、物流信息、活动规则
- 收集历史数据：好评、中评、差评案例
- 收集客户话术：优秀话术、红线禁忌

**步骤3：系统对接**
- 对接店铺后台：千牛、京麦、拼多多商家后台等
- 配置自动回复、快捷短语
- 配置消息分流、权限设置
- 对接数据统计系统

### 2.3 服务启动（T日）

**步骤1：启动会议**
- 项目启动会议，明确各方分工
- 确认服务范围、KPI指标、沟通机制
- 确定应急联系人

**步骤2：试运营**
- 试运营1-3天，客服主管全程指导
- 及时发现问题、解决问题
- 收集客户反馈，优化服务

**步骤3：正式运营**
- 正式启动服务
- 每日监控数据
- 第一周每日复盘

## 人员招聘与培训SOP

### 3.1 客服招聘标准

**3.1.1 基本要求**
- 大专及以上学历，专业不限
- 年龄18-35岁
- 打字速度≥60字/分钟
- 具备良好的沟通能力、服务意识
- 有电商客服经验者优先

**3.1.2 能力要求**
- 熟悉各大电商平台规则
- 具备问题解决能力
- 具备情绪管理能力
- 具备团队协作能力

### 3.2 招聘流程

**步骤1：简历筛选**
- 筛选符合基本要求的简历
- 优先考虑有相关经验的候选人

**步骤2：初试**
- 打字速度测试（60字/分钟合格）
- 结构化面试：沟通能力、服务意识、应变能力
- 平台知识测试

**步骤3：复试**
- 情景模拟：处理客户咨询、投诉
- 综合素质评估
- 确定录用

**步骤4：入职**
- 办理入职手续
- 签订劳动合同、保密协议
- 发放工牌、设备

### 3.3 培训体系

**3.3.1 新员工培训（14天）**

**第1-2天：基础培训**
- 公司介绍、规章制度
- 电商行业基础知识
- 客服职业素养
- 服务心态与沟通技巧
- 考核：书面测试80分合格

**第3-5天：平台培训**
- 淘宝/天猫平台规则
- 京东平台规则
- 拼多多平台规则
- 抖音平台规则
- 平台后台操作
- 考核：操作考核90分合格

**第6-8天：产品培训**
- 产品知识学习
- 商品参数、规格、功能
- 常见问题解答
- 竞品了解
- 考核：产品知识测试85分合格

**第9-11天：话术培训**
- 售前话术学习
- 售后话术学习
- 投诉处理话术
- 催付话术
- 考核：情景模拟90分合格

**第12-13天：实操培训**
- 老带新实操
- 客服主管指导
- 试接待客户（老带新）
- 每日复盘

**第14天：考核认证**
- 综合考核：理论+实操
- 85分以上合格，正式上岗
- 不合格延长培训3天

**3.3.2 在岗培训（持续）**

**每周培训**
- 每周1次业务培训（1小时）
- 学习新商品知识
- 学习新活动规则
- 优秀案例分享
- 问题案例分析

**每月培训**
- 每月1次技能提升培训（2小时）
- 沟通技巧提升
- 投诉处理进阶
- 数据分析能力

**3.3.3 培训记录**
- 建立员工培训档案
- 记录每次培训内容、考核结果
- 培训不合格安排补考

## 日常运营SOP

### 4.1 班前准备（提前15分钟）

**步骤1：设备检查**
- 检查电脑、网络、耳机是否正常
- 检查系统登录是否正常
- 检查后台是否正常

**步骤2：信息同步**
- 查看早会纪要
- 查看最新通知
- 了解库存变化
- 了解活动信息

**步骤3：状态调整**
- 调整心态，准备进入工作状态
- 登录客服系统
- 设置在线状态

### 4.2 日常接待流程

**4.2.1 售前咨询处理流程**

**步骤1：及时响应**
- 3分钟内响应客户咨询
- 首句问候："亲，您好！欢迎光临~ 请问有什么可以帮助您的吗？😊"
- 响应超时："非常抱歉让您久等了！请问有什么可以帮助您的？"

**步骤2：需求了解**
- 主动了解客户需求
- 引导客户说出具体问题
- 认真聆听，不要打断
- 确认客户需求："您是想了解XX对吗？"

**步骤3：产品介绍**
- 针对客户需求介绍产品
- 突出产品卖点
- 解答客户疑问
- 提供专业建议
- 不夸大宣传，实事求是

**步骤4：促成转化**
- 主动引导下单
- 告知活动优惠
- 告知发货时间
- 解答购买顾虑
- 催付："亲，您还在吗？喜欢的话可以尽快下单哦，我们今天就能发货呢~"

**步骤5：告别与跟进**
- 确认客户无其他问题
- 礼貌告别："感谢您的光临！有问题随时联系我们哦~ 祝您生活愉快！😊"
- 如有需要，跟进后续情况

**4.2.2 售后问题处理流程**

**步骤1：及时响应**
- 3分钟内响应售后咨询
- 首句问候："亲，您好！很抱歉给您带来了不好的体验，请问有什么问题呢？"
- 安抚客户情绪

**步骤2：问题了解**
- 耐心倾听客户问题
- 详细了解情况
- 要求客户提供图片/视频证据
- 确认问题："您说的是这个问题对吗？"

**步骤3：解决方案提供**
- 根据问题提供解决方案
- 参考退换货政策
- 提供2-3个方案供客户选择
- 不推诿责任

**步骤4：执行方案**
- 确认客户选择的方案
- 立即执行：退款/换货/补发/补偿
- 告知处理时效："我们会在24小时内为您处理，请您耐心等待~"

**步骤5：跟进回访**
- 跟进处理进度
- 确认客户是否收到退款/换货
- 确认客户是否满意
- 处理中差评："亲，非常抱歉给您带来了不好的体验，请问我们的处理您还满意吗？如果满意的话，可以麻烦您修改一下评价吗？非常感谢！😊"

**4.2.3 投诉处理流程**

**步骤1：升级处理**
- 遇到客户投诉，立即升级给客服主管
- 客服主管10分钟内介入

**步骤2：情绪安抚**
- 认真倾听客户投诉
- 真诚道歉
- 表达理解："我非常理解您现在的心情"

**步骤3：问题调查**
- 详细了解情况
- 核实事实真相
- 查找问题原因

**步骤4：解决方案**
- 与客户协商解决方案
- 在权限范围内满足客户合理诉求
- 超出权限上报项目经理

**步骤5：跟进确认**
- 跟进处理结果
- 确认客户满意
- 总结经验教训

### 4.3 班后整理（下班前10分钟）

**步骤1：工单处理**
- 处理完所有待办工单
- 交接未完成事项

**步骤2：数据记录**
- 记录当日接待数据
- 记录典型案例

**步骤3：系统退出**
- 保存所有数据
- 安全退出系统
- 关闭设备

## 客服话术库SOP

### 5.1 话术库结构

**5.1.1 分类方式**
- 按平台分类：淘宝、京东、拼多多、抖音等
- 按场景分类：售前、售中、售后、投诉
- 按品类分类：服装、3C、家居、食品等

**5.1.2 话术模板**
每个话术模板包含：
- 场景描述
- 标准话术
- 注意事项
- 变体话术

### 5.2 售前话术库

**5.2.1 问候与开场**

**场景1：客户首次咨询**
```
亲，您好！欢迎光临【店铺名】~ 请问有什么可以帮助您的吗？😊
```

**场景2：客户询问商品**
```
亲，这款商品非常不错呢！它的特点是【产品卖点1】、【产品卖点2】、【产品卖点3】，很多客户反馈都很好哦~
```

**场景3：客户询问尺码**
```
亲，这是我们的尺码表，您可以参考一下~ 【发送尺码表】
您平时穿什么尺码呢？我可以帮您推荐一下哦~
```

**5.2.2 产品咨询**

**场景1：询问功能**
```
亲，这款产品的主要功能是【功能描述】，具体操作是【操作说明】~
我给您发个使用视频您看看：【视频链接】
```

**场景2：询问质量**
```
亲，您放心哦！我们的产品都是正品，质量有保障的~
我们支持【7天无理由退换/30天包换/1年保修】，有任何问题随时联系我们~
```

**5.2.3 价格与优惠**

**场景1：询问价格**
```
亲，这款商品现在的价格是【价格】哦~
现在我们有【活动名称】活动，下单可以享受【优惠内容】，非常划算呢！
```

**场景2：议价**
```
亲，非常抱歉，我们的价格已经是最优惠的了呢~
不过现在下单我们可以赠送【赠品】，还是很划算的哦~
```

**5.2.4 物流与发货**

**场景1：询问发货时间**
```
亲，我们一般在【24小时/48小时】内发货哦~
下单后我们会尽快为您安排发出的！
```

**场景2：询问快递**
```
亲，我们默认发【快递名称】哦~
如果您需要指定快递，可以补运费发【快递名称】呢~
```

**5.2.5 催付话术**

**场景1：拍下未付款**
```
亲，您拍下的【商品】还没有付款哦~
现在库存紧张，付款后我们会尽快为您安排发货呢~
喜欢的话记得尽快付款哦！😊
```

**场景2：犹豫未下单**
```
亲，您还在考虑吗？这款商品真的很不错呢~
现在下单还可以享受【优惠】，非常划算哦！
喜欢就不要错过了~😊
```

### 5.3 售后话术库

**5.3.1 退换货**

**场景1：退换货申请**
```
亲，好的~ 我这边帮您处理退换货申请哦~
请您先申请退款/退货退款，原因选择【原因】
我们会尽快为您审核通过的~
```

**场景2：退换货地址**
```
亲，这是我们的退货地址哦：
收货人：【姓名】
电话：【电话】
地址：【详细地址】
请您寄回时在包裹里放个小纸条，写上您的订单号和退换货原因，方便我们处理哦~
```

**5.3.2 质量问题**

**场景1：商品有质量问题**
```
亲，非常抱歉给您带来了不好的体验！
请问您可以提供一下商品问题的图片或视频吗？
这边确认后会为您妥善处理的~
```

**场景2：解决方案**
```
亲，非常抱歉！我们可以为您【退款/换货/补发/补偿】，您看哪种方案比较合适呢？
```

**5.3.3 中差评处理**

**场景1：收到差评**
```
亲，非常抱歉给您带来了不好的体验！
请问您是遇到了什么问题呢？我们非常希望能为您解决~
您看我们【解决方案】，您能接受吗？
如果可以的话，麻烦您帮忙修改一下评价可以吗？非常感谢！😊
```

### 5.4 红线禁忌

**绝对禁止：**
- 与客户争吵、顶撞客户
- 对客户使用不礼貌语言
- 承诺无法兑现的事情
- 泄露客户隐私信息
- 贬低同行、竞品
- 违反平台规则

## 质检管理SOP

### 6.1 质检标准

**6.1.1 评分维度**

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 响应速度 | 20% | 3分钟内回复满分，超时扣分 |
| 服务态度 | 25% | 礼貌、耐心、热情，使用表情符号 |
| 专业程度 | 25% | 准确解答问题，产品知识熟悉 |
| 问题解决 | 20% | 首次解决，客户满意 |
| 合规性 | 10% | 符合平台规则，不违反红线 |

**6.1.2 评分等级**
- 90-100分：优秀
- 80-89分：良好
- 70-79分：合格
- 70分以下：不合格

### 6.2 质检流程

**6.2.1 日常质检**

**步骤1：抽样**
- 每位客服每日抽样≥10条对话
- 重点抽样：售后、投诉、转化相关对话

**步骤2：评分**
- 按照质检标准评分
- 记录问题点
- 填写《质检记录表》（见附录B）

**步骤3：反馈**
- 当日将质检结果反馈给客服
- 与客服沟通问题点
- 辅导改进

**6.2.2 专项质检**

**大促前质检**
- 大促前1周进行专项质检
- 检查活动话术掌握情况
- 检查流程熟练度

**投诉专项质检**
- 所有投诉对话100%质检
- 分析投诉原因
- 总结改进措施

### 6.3 质检数据应用

**6.3.1 个人改进**
- 针对个人问题制定改进计划
- 客服主管辅导改进
- 跟踪改进效果

**6.3.2 团队优化**
- 分析团队共性问题
- 开展针对性培训
- 优化话术库
- 优化流程

**6.3.3 绩效挂钩**
- 质检成绩与绩效挂钩
- 优秀奖励，不合格处罚

## 数据报表与复盘SOP

### 7.1 数据指标体系

**7.1.1 核心指标**

| 指标 | 定义 | 计算公式 |
|------|------|----------|
| 接待量 | 当日接待客户数 | 系统统计 |

| 咨询转化率 | 咨询后下单的比例 | 下单人数÷咨询人数 |
| 客单价 | 平均订单金额 | 总销售额÷订单数 |
| 客户满意度 | 客户评价分数 | 评价总分÷评价数 |
| 首次解决率 | 首次解决问题的比例 | 首次解决数÷问题总数 |

## 应急预案SOP

### 8.1 常见应急预案

**8.1.1 大促应急预案**

**场景：618、双11等大促期间**

**预案：**
- 提前1个月准备
- 增加客服人员配置（2-3倍）
- 提前储备兼职客服
- 优化自动回复、快捷短语
- 制定大促专用话术
- 延长服务时间（7×16小时或更长）
- 准备备用设备、备用网络
- 建立应急团队，随时补位

**8.1.2 系统故障应急预案**

**场景：客服系统、后台故障**

**预案：**
- 立即通知技术部门
- 使用备用系统（如手机APP）
- 使用电话、短信联系客户
- 在店铺首页挂公告，说明情况
- 记录问题，后续跟进
- 故障恢复后优先处理积压消息

**8.1.3 人员流失应急预案**

**场景：客服突然离职**

**预案：**
- 建立人才储备库
- 客服主管立即顶岗
- 尽快招聘补充
- 交接工作要完整（3天交接期）
- 培训新员工快速上岗

**8.1.4 舆情危机应急预案**

**场景：出现大量差评、负面评价**

**预案：**
- 立即上报项目经理
- 分析舆情原因
- 制定应对方案
- 积极联系客户，解决问题
- 发布官方声明（如需要）
- 监控舆情发展
- 总结经验教训

### 8.2 应急响应流程

**步骤1：发现问题**
- 发现应急情况

**步骤2：上报**
- 立即上报客服主管
- 客服主管上报项目经理
- 项目经理上报客户

**步骤3：启动预案**
- 启动相应应急预案
- 执行应急措施

**步骤4：解决问题**
- 快速解决问题
- 减少影响

**步骤5：总结复盘**
- 总结经验教训
- 优化预案
