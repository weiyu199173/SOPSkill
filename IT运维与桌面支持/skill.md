# IT运维与桌面支持 Skill

## 核心能力

### 1. 日常巡检与监控
- 服务器 CPU、内存、磁盘、带宽 7x24 监控
- 网站/APP 响应时间、错误率、流量实时检查
- 日志分析与异常检测
- 预防性维护与隐患排查

### 2. 工单处理
- 开通/禁用员工账号、权限快速调整
- 软件安装、环境配置、电脑/网络问题解决
- 数据查询、备份恢复、日志导出
- 平均响应时间 ≤ 15分钟

### 3. 版本发布与变更管理
- 发布前检查：配置、依赖、备份、回滚方案
- 灰度发布：先放少量流量，观察无问题再全量
- 发布中监控：错误率、响应时间、流量变化
- 变更成功率 ≥ 99.9%

## SOP 流程

### 故障处理流程
1. 故障发现 → 监控告警触发
2. 故障分级 → P0/P1/P2/P3 等级判定
3. 快速响应 → 5分钟内响应 P0 故障
4. 根因分析 → 5Why 分析法
5. 问题解决 → 修复并验证
6. 复盘总结 → 输出故障报告

### 变更发布流程
1. 变更申请 → 填写变更单
2. 技术评审 → 评估风险与方案
3. 准备阶段 → 配置、备份、回滚方案
4. 灰度发布 → 小范围验证
5. 全量发布 → 全面推广
6. 监控验证 → 持续观察 24小时

## 常用命令速查

### Linux 系统管理
```bash
# 系统资源监控
top
htop
vmstat 1

# 磁盘空间检查
df -h
du -sh *

# 日志查看
tail -f /var/log/syslog
journalctl -xe

# 服务管理
systemctl status nginx
systemctl restart nginx
```

### 网络诊断
```bash
# 网络连通性
ping 8.8.8.8
traceroute example.com

# 端口检查
telnet host port
nc -zv host port

# 网络流量
iftop
nethogs
```

## 故障排查清单

### 服务不可用排查
1. 检查服务状态 `systemctl status`
2. 查看服务日志
3. 检查端口是否被占用 `netstat -tlnp`
4. 检查磁盘空间 `df -h`
5. 检查内存使用 `free -h`
6. 检查依赖服务是否正常

### 网络问题排查
1. ping 测试连通性
2. traceroute 追踪路由
3. 检查防火墙规则
4. 检查 DNS 解析
5. 检查负载均衡配置

## 安全最佳实践

- 定期更新系统补丁
- 最小权限原则配置用户权限
- 启用 SSH 密钥登录，禁用密码登录
- 配置防火墙，只开放必要端口
- 定期备份关键数据
- 启用操作审计日志
