# 软件测试与质量保证 Skill

## 核心能力

### 1. 测试规划
- 需求分析与测试策略制定
- 高质量测试用例设计
- 完善的测试环境准备
- 测试计划专业撰写

### 2. 测试执行
- 全面功能测试
- 专业性能测试
- 深度安全测试
- 多端兼容性测试

### 3. 缺陷管理
- 缺陷发现与精准记录
- 缺陷跟踪与验证
- 专业测试报告输出
- 线上问题快速响应

## SOP 流程

### 测试流程
1. 需求分析 → 理解产品需求
2. 测试计划 → 制定测试策略
3. 用例设计 → 设计测试用例
4. 环境准备 → 搭建测试环境
5. 测试执行 → 执行测试用例
6. 缺陷管理 → 跟踪缺陷修复
7. 测试报告 → 输出测试总结

### 缺陷处理流程
1. 缺陷发现 → 复现问题步骤
2. 缺陷记录 → 填写缺陷单
3. 缺陷评审 → 确认缺陷有效性
4. 缺陷修复 → 开发修复问题
5. 缺陷验证 → 测试验证修复
6. 缺陷关闭 → 确认修复完成

## 测试用例设计方法

### 等价类划分
- 有效等价类：合理的输入数据
- 无效等价类：不合理的输入数据
- 每个等价类选取代表性数据

### 边界值分析
- 最小值、略大于最小值
- 最大值、略小于最大值
- 正常值、边界值组合

### 错误推测法
- 基于经验推测可能的错误
- 针对常见错误设计用例
- 历史问题重点覆盖

### 场景法
- 模拟用户实际使用场景
- 端到端业务流程测试
- 异常场景处理测试

## 测试类型详解

### 功能测试
- 单元测试：代码级功能验证
- 集成测试：模块间接口验证
- 系统测试：全系统功能验证
- 验收测试：用户验收验证

### 性能测试
- 负载测试：正常负载下表现
- 压力测试：极限负载下表现
- 稳定性测试：长时间运行稳定性
- 并发测试：多用户并发操作

### 安全测试
- 渗透测试：模拟攻击测试
- 权限测试：权限控制验证
- 数据安全：数据加密验证
- 漏洞扫描：安全漏洞检测

## 测试用例模板
```markdown
# 测试用例
用例编号：TC-001
用例标题：用户登录功能验证
优先级：P0
前置条件：用户账号已注册
测试步骤：
  1. 打开登录页面
  2. 输入正确用户名
  3. 输入正确密码
  4. 点击登录按钮
预期结果：
  1. 登录成功
  2. 跳转到首页
  3. 显示用户信息
实际结果：待测试
测试状态：待执行
```
