## SOP概述

### 1.1 目的
本SOP旨在规范数据标注与AI训练数据服务的全流程管理，确保标注质量稳定、高效，为客户提供高质量的AI训练数据，帮助客户提升AI模型的准确率和可靠性。

### 1.2 适用范围
本SOP适用于各类数据标注服务，包括但不限于：图像标注（物体检测、语义分割、关键点标注、图像分类、OCR标注）、文本标注（实体识别、情感分析、意图分类、文本摘要、对话数据标注）、音频/视频标注（语音转写、场景识别、动作标注、多模态标注）。

### 1.3 术语定义
| 术语 | 定义 |
|------|------|
| SOP | 标准作业程序 |
| KPI | 关键绩效指标 |
| 标注 | 对原始数据添加标签、注释的过程 |
| 边界框 | 图像中用矩形框标注物体的方式 |
| 语义分割 | 对图像中每个像素进行分类标注 |
| 关键点标注 | 标注图像中物体的关键坐标点 |
| 实体识别 | 标注文本中的命名实体 |
| 初标 | 第一轮标注 |
| 复检 | 对初标结果的复核检查 |
| 抽检 | 随机抽取样本进行质量检查 |
| IOU | 交并比，用于评估边界框标注质量 |
| 一致性 | 多个标注员标注同一数据的一致程度 |
| 数据脱敏 | 对敏感信息进行处理，保护隐私 |

### 1.4 核心KPI指标
| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 标注准确率 | ≥ 98.5% | 质量审核统计 |
| 交付准时率 | ≥ 99.5% | 项目计划对比 |
| 返工率 | < 1.5% | 返工数据比例 |
| 一致性准确率 | ≥ 99% | 多人标注一致性 |
| 数据完整性 | 100% | 数据检查 |
| 客户满意度 | ≥ 4.8分（5分制） | 客户评价统计 |

## 项目对接与需求分析

### 2.1 客户对接流程

#### 2.1.1 初步接触阶段
**步骤1：客户需求收集**
- 填写《数据标注需求调研表》（见附录A）
- 了解项目背景、业务目标、AI模型应用场景
- 明确标注类型、数据格式、数据量
- 收集项目文档：需求文档、标注示例、验收标准

**步骤2：技术方案评估**
- 技术负责人评估技术可行性
- 确定标注难度、标注复杂度
- 评估所需资源（标注员、审核员、工具）
- 制定初步标注方案和报价

#### 2.1.2 需求确认阶段
**步骤1：需求评审会议**
- 与客户召开需求评审会议
- 详细讨论标注规范、标注粒度
- 确认验收标准、SLA指标
- 明确沟通机制、交付物要求

**步骤2：需求文档梳理**
- 梳理并完善标注规范文档
- 识别标注中的模糊点、边界案例
- 与客户确认需求变更流程
- 形成最终《标注需求说明书》

### 2.2 标注方案制定

#### 2.2.1 方案编制
**步骤1：确定标注策略**
- 根据数据特点选择标注方法
- 确定标注工具和标注粒度
- 制定标注优先级策略
- 确定质量审核策略

**步骤2：制定详细方案**
- 编写《数据标注方案书》
- 明确标注范围、标注目标
- 制定标注进度计划
- 确定标注团队配置
- 制定质量控制措施
- 制定风险应对措施

#### 2.2.2 方案评审
**步骤1：内部评审**
- 标注团队内部评审方案
- 技术负责人审核技术方案
- 项目经理审核资源和进度
- 修改完善方案

**步骤2：客户确认**
- 提交方案给客户评审
- 收集客户反馈意见
- 调整优化方案
- 双方签字确认

### 2.3 合同签订与项目启动

#### 2.3.1 合同签订
**步骤1：合同准备**
- 准备《数据标注服务合同》
- 明确服务范围、价格、付款方式
- 明确SLA指标、违约责任
- 明确知识产权、保密条款

**步骤2：合同签署**
- 双方确认合同条款
- 正式签署合同
- 客户支付首期款项
- 合同归档

#### 2.3.2 项目启动
**步骤1：项目启动会议**
- 召开项目启动会
- 介绍项目团队成员
- 明确项目目标、时间节点
- 确定沟通机制、汇报机制
- 明确各方责任分工

**步骤2：项目立项**
- 在项目管理平台创建项目
- 分配项目权限
- 初始化标注管理平台
- 建立项目文档库
- 创建标注规范库

### 2.4 标注团队组建

#### 2.4.1 团队配置
根据项目规模和复杂度，配置以下角色：
- 项目经理：1人，负责项目整体管理
- 标注负责人：1人，负责标注技术和团队管理
- 标注员：5-50人，根据项目规模
- 审核员：2-10人，负责质量审核
- 技术支持：1-2人，负责工具支持

#### 2.4.2 团队培训
**步骤1：项目培训**
- 讲解项目背景、业务知识
- 培训标注规范、标注工具
- 讲解标注要求、验收标准
- 明确标注流程、质量要求

**步骤2：考核认证**
- 标注规范考试（90分合格）
- 实操考核（100个样本标注，准确率≥98%合格）
- 考核合格方可上岗

### 2.5 项目计划制定

#### 2.5.1 进度计划
**步骤1：WBS分解**
- 将标注工作分解为任务
- 估算每个任务的工作量
- 确定任务依赖关系
- 形成WBS工作分解结构

**步骤2：甘特图编制**
- 使用项目管理工具编制甘特图
- 分配任务负责人
- 设置里程碑节点
- 确定关键路径

#### 2.5.2 资源计划
**步骤1：人力资源计划**
- 确定各阶段人力需求
- 制定人员投入计划
- 识别人员缺口和储备

**步骤2：工具资源计划**
- 确定所需标注工具
- 准备标注平台
- 申请软件License
- 准备标注环境

## 数据采集与预处理

### 3.1 数据采集规范

#### 3.1.1 数据接收流程
**步骤1：数据接收检查**
- 检查数据格式是否符合要求
- 检查数据完整性
- 检查数据大小
- 记录数据接收情况

**步骤2：数据登记**
- 数据编号
- 数据分类
- 数据统计
- 数据入库

#### 3.1.2 数据质量评估
**评估指标**：
- 数据完整性：无缺失、无损坏
- 数据清晰度：图像清晰、文本可读
- 数据多样性：覆盖各种场景
- 数据平衡性：类别分布合理

### 3.2 数据清洗流程

#### 3.2.1 数据去重
**方法1：哈希去重**
- 计算数据哈希值
- 比较哈希值
- 删除重复数据

**方法2：内容比对**
- 图像相似度比对
- 文本相似度比对
- 删除相似数据

#### 3.2.2 数据修复
**修复内容**：
- 图像修复：裁剪、旋转、亮度调整
- 文本修复：格式统一、编码统一
- 删除无效数据

#### 3.2.3 数据筛选
**筛选标准**：
- 数据质量
- 数据相关性
- 数据有效性

### 3.3 数据格式转换

#### 3.3.1 格式标准化
**图像格式**：
- 统一转换为JPG/PNG
- 统一分辨率
- 统一尺寸

**文本格式**：
- 统一编码UTF-8
- 统一换行符
- 统一文件扩展名

#### 3.3.2 数据分割
**分割原则**：
- 按批次分割
- 按类别分割
- 按难度分割
- 训练集/验证集/测试集分割

### 3.4 隐私数据脱敏处理

#### 3.4.1 敏感信息识别
**识别内容**：
- 人脸信息
- 身份证号
- 手机号
- 邮箱
- 地址
- 其他个人信息

#### 3.4.2 脱敏方法
**图像脱敏**：
- 人脸模糊
- 马赛克处理
- 像素化

**文本脱敏**：
- 替换为占位符
- 加密处理
- 删除敏感信息

### 3.5 数据存储与备份

#### 3.5.1 数据存储
**存储规范**：
- 分类存储
- 目录结构规范
- 文件命名规范
- 元数据记录

#### 3.5.2 数据备份
**备份策略**：
- 本地备份
- 云端备份
- 定期备份
- 备份验证

## 图像标注作业流程

### 5.1 物体检测标注

#### 5.1.1 标注规则
**边界框规则**：
- 边界框紧贴物体边缘
- 边界框包含整个物体
- 遮挡物体标注可见部分
- 小物体（<32x32像素）也需标注
- 同类物体标注风格一致
- 重叠物体分别标注

#### 5.1.2 标注步骤
**步骤1：数据预览**
- 打开标注工具
- 加载待标注图片
- 预览图片内容

**步骤2：标注执行**
- 识别目标物体
- 绘制边界框
- 选择物体类别
- 标注物体属性（如需要）

**步骤3：自我检查**
- 检查边界框位置
- 检查类别是否正确
- 检查是否遗漏物体
- 检查是否重复标注

**步骤4：提交审核**
- 保存标注结果
- 提交到审核队列

#### 5.1.3 标注示例
```
标注示例1：人物标注
- 边界框：从头顶到脚底
- 类别：person
- 属性：姿态（站立/坐姿/躺卧）

标注示例2：车辆标注
- 边界框：包含整个车辆
- 类别：car
- 属性：颜色（红色/蓝色/...）
```

### 5.2 语义分割标注

#### 5.2.1 标注规则
**分割规则**：
- 每个像素都要有标签
- 边界准确
- 相邻区域标注正确
- 遮挡区域标注可见部分

#### 5.2.2 标注步骤
**步骤1：数据预览**
- 打开分割标注工具
- 加载图片

**步骤2：区域标注**
- 选择标注工具（画笔/多边形/智能分割）
- 标注每个区域
- 选择区域类别

**步骤3：检查优化**
- 检查边界是否准确
- 检查区域是否完整
- 优化标注细节

**步骤4：提交审核**

### 5.3 关键点标注

#### 5.3.1 标注规则
**关键点规则**：
- 关键点位置准确
- 关键点数量正确
- 关键点顺序一致
- 遮挡关键点标注估计位置

#### 5.3.2 标注步骤
**步骤1：加载骨架模板**
- 选择关键点模板
- 加载到图片上

**步骤2：调整关键点**
- 逐个调整关键点位置
- 标记可见/遮挡状态

**步骤3：检查验证**
- 检查关键点位置
- 检查关键点数量
- 检查关键点连接关系

**步骤4：提交审核**

#### 5.3.3 关键点示例
```
人脸关键点标注（68点）：
- 轮廓点
- 眼睛
- 鼻子
- 嘴巴
- 眉毛

人体关键点标注（17点）：
- 鼻子
- 眼睛
- 耳朵
- 肩膀
- 手肘
- 手腕
- 髋部
- 膝盖
- 脚踝
```

### 5.4 图像分类标注

#### 5.4.1 分类规则
**分类规则**：
- 类别选择准确
- 单标签/多标签明确
- 边界案例按规则处理

#### 5.4.2 标注步骤
**步骤1：理解分类标准**
- 阅读分类规范
- 理解每个类别定义

**步骤2：图像分类**
- 观察图像内容
- 选择对应类别
- 标记不确定案例

**步骤3：检查确认**
- 检查类别是否正确
- 检查是否漏标

**步骤4：提交审核**

### 5.5 OCR文本识别标注

#### 5.5.1 标注规则
**OCR规则**：
- 文本转录准确
- 标点符号正确
- 特殊字符处理规范
- 多语言标注规范

#### 5.5.2 标注步骤
**步骤1：文本区域识别**
- 识别图片中的文本区域
- 绘制文本框

**步骤2：文本转录**
- 逐字转录文本
- 检查转录内容

**步骤3：文本属性标注**
- 标注文本语言
- 标注文本类型（印刷体/手写体）

**步骤4：检查提交**

### 5.6 边界案例处理规则

#### 5.6.1 常见边界案例
| 案例类型 | 处理规则 |
|----------|----------|
| 遮挡物体 | 标注可见部分 |
| 部分可见物体 | 标注可见部分 |
| 模糊物体 | 能识别则标注，不能识别标记为不确定 |
| 小物体 | 小于32x32像素也标注 |
| 重叠物体 | 分别标注 |
| 同类物体密集 | 逐个标注 |
| 背景干扰 | 排除背景，只标注目标 |

#### 5.6.2 不确定案例处理
**处理流程**：
1. 标记为不确定
2. 记录问题
3. 提交审核
4. 等待确认

## 文本标注作业流程

### 6.1 实体识别标注

#### 6.1.1 实体类型
**常见实体类型**：
- 人名
- 地名
- 组织机构名
- 时间
- 日期
- 金额
- 产品名
- 其他自定义实体

#### 6.1.2 标注规则
**标注规则**：
- 实体边界准确
- 实体类型正确
- 完整实体标注
- 重叠实体分别标注

#### 6.1.3 标注步骤
**步骤1：文本阅读**
- 阅读文本内容
- 理解文本上下文

**步骤2：实体识别**
- 识别实体
- 选择实体类型
- 标注实体

**步骤3：检查确认**
- 检查实体边界
- 检查实体类型
- 检查是否遗漏

**步骤4：提交审核**

#### 6.1.4 标注示例
```
示例文本：张三于2026年4月20日在北京科技大学参加会议。

标注结果：
- 张三：人名
- 2026年4月20日：日期
- 北京科技大学：组织机构名
```

- 实体边界准确率 ≥ 98.5%

### 附录E：常见问题解答（FAQ）

**Q1：标注过程中需求变更怎么办？**
A：按照需求变更流程处理，评估变更影响，调整项目计划，更新标注规范，必要时重新培训。

**Q2：标注员标注质量不达标怎么办？**
A：分析原因，重新培训，加强审核，必要时调整标注员任务。

**Q3：项目时间不够怎么办？**
A：增加人员投入，延长工作时间，优化标注流程，与客户沟通调整范围或时间。

**Q4：标注过程中遇到不确定的案例怎么办？**
A：标记为不确定，记录问题，提交审核，等待确认。
